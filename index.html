Geodatabases

A short introduction to geodatabases and ArcGIS technology

Geodatabases are data structures specializing in storing location-based physical phenomena or statistics. They are a specific applicaiton of GIS.

GIS stands for Georgraphic Information Science or Georgraphic Information System. The system is a type of Spatial Data infrastructure. Combined with AI, GIS's can power intelligent decision-making platforms.

Geodatabases are widely used, in fields like real estate, public health (epidemiology), crime mapping, national defense, sustainable development, natural resources, climatology, location based services.

Architecture Basics

Geodatabases store data using primarily a SQL DBMS, following an object-relational model. Object-relational models utilize the familiar SQL relational model, but allow for creation of user-defined data types.

Geodatabases are composed of system tables and user data. Custom data types are formatted using data in system tables (XML) and combined with user data for end presentation.

User data is defined in three varieties.

Tables are the traditional structure, holding rows of data with corresponding field types

Feature classes are tables, but include shape data as well. Shape data can hold point, line, or polgyon geometry.

Raster datasets contain raster information, raw imagery or geographic data that is indexed with table metadata.

More dataset types can be defined per geodatabase.

Tables and Feature Classes

Tables can hold all the same data types as standard SQL. The BLOB type is often used for CAD geometry and symbols. GUIDs (Globablly Unique IDentifiers are often used to synchronize/avoid conflict with geodatabase data across organization.

Feature classes define geometric elements (mainly vectorized) in homogenous collections.

Points, lines, and polygons are in traditional spatial databases, but geodatabases add annotation, dimension, multipoint, and multipatch. See field descriptions here.

Feature classes can be extended to track more complex data like topology, network data.

Features of these two dataset structures include the ability to define attribute domains, relationship classes, and subtypes.

Features

Features are described by feature type, discrete parts, curves, vertex coordinates, linear measures

Note: Edges can be straight, or follow a circular or elliptical arc, or follow a bezier curve.

Linear measures allow linear referencing: an indicator that describes the feature without giving coordinates for the location. Also known as m-values, linear measures allow for benchmarking, or parametrization, of edges/paths, based on arbitrary, useful parameters.

A path/feature can have multiple sets of m-values applied to it. This technique allows for dynamic segmentation, which allows data-dense descriptions of features with only one copy of physical location.

Read more about dynamic segmentation here.

Raster Basics







Raster data usually comes in the form of images. Mosaicking is the technique of breaking raster data into chunks and tablifying for quicker manipulation. Raster catalogs organize images in time/location.

In processing raster data, users deal with resolution and tolerance. Resolution offers a measure of precision usually given by the raw data, and tolerance is adjusted to automatically create feature classes based on image processing techniques.
